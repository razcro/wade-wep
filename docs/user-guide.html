User Guide: Managing Newspaper Article Provenance

Authors

  • Team rc-af

Abstract

This user guide provides comprehensive instructions for using the NewsProvenience platform, a semantic web application for managing newspaper article provenance. The guide covers basic navigation, search functionality, article management, SPARQL queries, and visualization features. Three detailed case studies demonstrate real-world usage scenarios including finding fresh editorials, filtering articles by language and word count, and discovering investigative journalism content.

1. Introduction

1.1 What is NewsProvenience?

NewsProvenience is a web-based platform designed to help journalists, researchers, and media professionals manage and query knowledge about newspaper articles and their provenance. The system provides:

  • Semantic Search - Find articles using SPARQL-powered queries
  • Provenance Tracking - Trace article origins and sources
  • Multi-language Support - Filter by language (EN, RO, ES, etc.)
  • Multimedia Content - Manage articles, podcasts, documentaries
  • Graph Visualization - Explore RDF relationships visually
  • Recommendations - Discover related content automatically

2.3 Article Detail Page

2.3.1 Overview Tab

Displays the complete article with semantic RDFa markup:

  • Full article headline and body text
  • Author information with links to author profile
  • Topic tags (SKOS concepts)
  • Associated media (images, audio player, video player)
  • Source URLs for provenance verification
  • QR code for sharing the article's semantic URI

2.3.2 Provenance Tab

The Provenance tab shows:

  • Activity - The ingestion process that created the article
  • Agent - System or person responsible for the ingestion
  • Sources Used - Original URLs and documents referenced
  • Attribution - Link to the article author
  • Timestamps - When the article was processed

2.3.3 Recommendations Tab

The recommendations are generated by a SPARQL query that finds articles sharing the same schema:about topics. Articles are ranked by the number of shared topics.

2.3.4 Graph Tab

The graph visualization shows:

  • Nodes - The article, author, topics, and linked resources
  • Edges - RDF predicates (schema:author, schema:about, prov:wasAttributedTo)
  • Interactive Features - Zoom, pan, node selection, and highlighting
Tip: Click on nodes in the graph to see their URIs. Double-click to expand the neighborhood and discover more related entities.

3.3 Example Queries

3.3.1 Find All Articles by a Specific Author

PREFIX schema: <http://schema.org/>

SELECT ?title ?date WHERE {
    ?article a schema:NewsArticle ;
             schema:headline ?title ;
             schema:author ?author ;
             schema:datePublished ?date .

    ?author schema:name "John Smith" .
}
ORDER BY DESC(?date)

3.3.2 Count Articles by Topic

PREFIX schema: <http://schema.org/>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>

SELECT ?topicName (COUNT(?article) as ?count) WHERE {
    ?article a schema:NewsArticle ;
             schema:about ?topic .

    ?topic skos:prefLabel ?topicName .
}
GROUP BY ?topicName
ORDER BY DESC(?count)

3.3.3 Find Articles with DBpedia Links

PREFIX schema: <http://schema.org/>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>

SELECT ?title ?dbpediaUri WHERE {
    ?article a schema:NewsArticle ;
             schema:headline ?title ;
             schema:about ?topic .

    ?topic skos:exactMatch ?dbpediaUri .
    FILTER(STRSTARTS(STR(?dbpediaUri), "http://dbpedia.org"))
}

4. Case Studies

Case Study 1: Finding Fresh Editorials on Technology

Scenario

Maria is a technology journalist who wants to find all recently published editorials about technology topics from the last month. She needs articles that are current and relevant to her upcoming news roundup.

Step-by-Step Solution

1Navigate to the Search Page

Maria clicks "Search" in the navigation bar to access the advanced search interface.

2Enter Search Criteria

Maria enters the following filters:

  • Query: "technology"
  • Media Type: Article (Editorial)
  • Date From: 2024-12-01

3Enable SPARQL Search

Maria toggles "Use SPARQL Search" to leverage the semantic query capabilities.

4Execute Search

Clicking "Search" executes the following SPARQL query:

PREFIX schema: <http://schema.org/>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>
PREFIX xsd: <http://www.w3.org/2001/XMLSchema#>

SELECT ?article ?title ?description ?date ?author WHERE {
    ?article a schema:NewsArticle ;
             schema:headline ?title ;
             schema:datePublished ?date ;
             schema:about ?topic .

    ?topic skos:prefLabel ?topicLabel .
    FILTER(CONTAINS(LCASE(?topicLabel), "technology"))

    OPTIONAL { ?article schema:description ?description }
    OPTIONAL {
        ?article schema:author ?authorRes .
        ?authorRes schema:name ?author .
    }

    FILTER(?date >= "2024-12-01T00:00:00"^^xsd:dateTime)
}
ORDER BY DESC(?date)
LIMIT 20

5Review Results

Maria reviews the search results showing recent technology editorials.

6Explore Article Details

Maria clicks on an article to view its full content, provenance, and related articles.

Result: Maria found 15 recent technology editorials published in the last month. She can now use these for her news roundup, with full provenance information available for fact-checking.

Case Study 2: Articles Under 4000 Words in English or Spanish About IT Contests

Scenario

Carlos is a content curator for an international tech magazine. He needs to find concise articles (under 4000 words) about international IT contests that are available in either English or Spanish for the magazine's bilingual readership.

Step-by-Step Solution

1Use the API Endpoint Directly

Carlos accesses the predefined query endpoint:

GET /api/sparql/examples/articles-by-language?lang1=en&lang2=es&maxWords=4000&topic=IT%20contest

2Alternative: Use Search Interface

Carlos can also use the Search page with these filters:

  • Query: "IT contest" or "programming competition"
  • Language: en (first search), es (second search)
  • Max Words: 4000

3SPARQL Query Executed

The system executes:

PREFIX schema: <http://schema.org/>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>

SELECT ?article ?title ?language ?wordCount ?topic WHERE {
    ?article a schema:NewsArticle ;
             schema:headline ?title ;
             schema:inLanguage ?language ;
             schema:wordCount ?wordCount .

    FILTER(?language IN ("en", "es"))
    FILTER(?wordCount < 4000)

    OPTIONAL {
        ?article schema:about ?topicRes .
        ?topicRes skos:prefLabel ?topic .
    }

    FILTER(
        CONTAINS(LCASE(?title), "it contest") ||
        CONTAINS(LCASE(?title), "programming") ||
        CONTAINS(LCASE(?topic), "competition")
    )
}
ORDER BY ?language ?wordCount

4Review and Export Results

Carlos reviews the bilingual results and exports selected articles.

5Export as JSON-LD

Carlos exports selected articles as JSON-LD for integration with the magazine's CMS:

GET /api/articles/{id}/export/jsonld
Result: Carlos found 8 articles meeting his criteria - 5 in English and 3 in Spanish, all under 4000 words and covering various international IT contests. The word count filter ensured all articles fit within the magazine's page limits.

Case Study 3: Discovering Investigations and Documentaries by Romanian Journalists

Scenario

Elena is a media researcher studying investigative journalism in Eastern Europe. She wants to compile a list of all investigations and documentaries produced by Romanian journalists that have been published on the web, including their titles and descriptions.

Step-by-Step Solution

1Access the Predefined Query

Elena uses the dedicated endpoint for Romanian investigations:

GET /api/sparql/examples/romanian-investigations

2Understand the Query

The endpoint executes this SPARQL query:

PREFIX schema: <http://schema.org/>
PREFIX prov: <http://www.w3.org/ns/prov#>

SELECT ?title ?description ?authorName ?publishDate WHERE {
    ?article a schema:NewsArticle ;
             schema:headline ?title ;
             schema:genre ?genre ;
             prov:wasAttributedTo ?author .

    ?author schema:name ?authorName ;
            schema:nationality "Romanian" .

    FILTER(?genre IN ("investigation", "documentary"))

    OPTIONAL { ?article schema:description ?description }
    OPTIONAL { ?article schema:datePublished ?publishDate }
}
ORDER BY DESC(?publishDate)

3View Results in the Interface

Elena accesses the SPARQL page and runs the query manually to see results in the UI.

4Examine Results

The results show investigations and documentaries with their metadata.

5Explore Individual Articles

Elena clicks on an investigation to view its full provenance chain.

6Visualize the Knowledge Graph

Elena uses the Graph tab to explore relationships between Romanian journalists and their investigative work.

7Export Data for Research

Elena exports the results as CSV for her research database:

POST /api/sparql?format=csv
Result: Elena discovered 12 investigations and 4 documentaries by Romanian journalists. The provenance information helped her verify the original sources, and the graph visualization revealed connections between journalists working on similar topics. She exported the data for further analysis in her research paper.

5.2 Embedded RDFa Markup

Every article page contains RDFa markup that can be extracted by semantic web crawlers and search engines: 

<article vocab="http://schema.org/" typeof="NewsArticle CreativeWork">
    <h1 property="headline">Article Title</h1>
    <span property="author" typeof="Person">
        <span property="name">Author Name</span>
    </span>
    <time property="datePublished">2024-12-15</time>
    <div property="articleBody">...</div>
</article>
Tip: Use Google's Rich Results Test or the W3C RDFa Validator to verify the embedded semantic markup on any article page.

5.3 Multimedia Content

NewsProvenience handles different media types with appropriate players:

  • Podcasts (Audio) - HTML5 audio player for podcast episodes with playback controls
  • Documentaries (Video) - HTML5 video player for documentary content with full-screen support

5.4 QR Code Sharing

Each article page includes a QR code that encodes the article's semantic URI. This enables:

  • Quick mobile access to article pages
  • Sharing provenance-verified content
  • Physical-digital linking for printed materials

6. Troubleshooting

6.1 Common Issues

Issue Possible Cause Solution
Search returns no results Filters too restrictive Broaden date range or remove language filter
SPARQL query error Syntax error in query Check PREFIX declarations and bracket matching
Graph won't load Too many triples Add LIMIT clause to reduce result size
Slow response Complex query or large dataset Add more specific filters, use LIMIT
Media won't play Unsupported format or broken URL Check original source URL availability

6.2 SPARQL Error Messages

Error: "Parse error"
Check your SPARQL syntax. Common issues include:
  • Missing PREFIX declarations
  • Unmatched curly braces { }
  • Missing periods at end of triple patterns
  • Invalid URI format (use <uri> or prefix:local)
Error: "Query timeout"
The query is taking too long. Try:
  • Adding a LIMIT clause
  • Being more specific with FILTER conditions
  • Removing expensive operations like COUNT or GROUP BY

8. Glossary

JSON-LD
JavaScript Object Notation for Linked Data - a method of encoding RDF data in JSON format
Provenance
Information about the origins, custody, and ownership of an artifact (article)
PROV-O
The PROV Ontology - W3C standard for representing provenance information
RDF
Resource Description Framework - a standard model for data interchange on the Web
RDFa
RDF in Attributes - a way to embed RDF data in HTML documents
SKOS
Simple Knowledge Organization System - W3C standard for representing taxonomies
SPARQL
SPARQL Protocol and RDF Query Language - query language for RDF data
Triple
The basic unit of RDF data: subject-predicate-object
URI
Uniform Resource Identifier - a unique identifier for resources
